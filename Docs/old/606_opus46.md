## The Problem This Document Solves

FreeServicesHub is a **distributed agent monitoring platform** — a web Hub that manages remote Windows Service agents. After a deep code exploration (performed by Opus sub-agent against the live workspace), the project turned out to be far more complete than the original documentation suggested. The Agent worker service runs, heartbeats flow, the dashboard renders live metrics, and the auth pipeline hashes tokens correctly.

But there's a gap. The system can **monitor** agents; it cannot **command** them. There is no mechanism for the Hub to say "Agent, go run this script" or "Agent, collect those logs and report back." The Agent heartbeats are one-way (Agent → Hub). The missing piece is a **Job Queue** — a bidirectional contract where the Hub queues work items and agents poll, execute, and report results.

### What Exists Today (The Before)

```ascii
  ┌─────────────────────────────────────────────────────────────────┐
  │                    CURRENT STATE                                │
  │                                                                 │
  │  Hub Web Server                        Agent Worker Service     │
  │  ══════════════                        ════════════════════     │
  │                                                                 │
  │  ✅ Agent Dashboard (live cards)       ✅ Heartbeat loop        │
  │  ✅ Agent Management (key gen)         ✅ System snapshots      │
  │  ✅ Agent Settings (inline edit)       ✅ SignalR connection    │
  │  ✅ Background Services (log view)     ✅ Registration flow     │
  │  ✅ ApiKeyMiddleware (SHA-256 auth)    ✅ Exponential backoff   │
  │  ✅ AgentMonitorService (stale poll)   ✅ Windows Service       │
  │  ✅ EF Models (Agent, Heartbeat,       ✅ appsettings config    │
  │     RegistrationKey, ApiClientToken)                            │
  │                                                                 │
  │  ❌ No Job Queue DTO                   ❌ No job fetch loop     │
  │  ❌ No job CRUD endpoints              ❌ No job execution      │
  │  ❌ No job status on dashboard         ❌ No job reporting      │
  │  ⚠️  AgentMonitorService broadcasts    ❌ No E2E test scripts   │
  │     to global group (tenant leak)                               │
  │                                                                 │
  │  DATA FLOW (one-way only):                                     │
  │  Agent ────heartbeat────► Hub ────SignalR────► Dashboard        │
  │                                                                 │
  │  MISSING DATA FLOW:                                             │
  │  Hub ────job queue────► Agent ────results────► Hub ──► Dashboard│
  └─────────────────────────────────────────────────────────────────┘
```

### What We're Building (The After)

The complete bidirectional command loop:

1. **Phase 1 — Data Contracts:** Define the `HubJob` entity through every solution layer (EFModels → DataObjects → DataAccess → Controller). Without this, there is no shared vocabulary between Hub and Agent for "what work needs to happen."

2. **Phase 2 — Agent Hardening:** Wire the Agent to poll for jobs, execute them locally, and report results. Also fix the one real security vulnerability found — `AgentMonitorService` broadcasting to a global SignalR group without tenant partitioning.

3. **Phase 3 — Dashboard & E2E:** Give users visibility into the job queue on the dashboard, add a verification step to the provisioning wizard, and create the first Playwright test scripts for the Hub's custom pages.

### Why This Matters

Without a job queue, the Hub is a **read-only monitoring dashboard** — useful, but it's half the product. The Agent runs on remote servers with elevated privileges (LocalSystem). The entire value proposition of FreeServicesHub is that you can **manage** those servers from one central web UI: restart services, collect logs, run maintenance scripts. The heartbeat pipeline proves the architecture works. The job queue completes it.

## Phase 1: Data Contracts

### Why Phase 1 Exists

The codebase has a complete data pipeline for **Agents** (register, heartbeat, monitor) but zero infrastructure for **Jobs**. When we explored the actual source, we found:

- `FreeServicesHub.App.DataObjects.Agents.cs` — Agent DTO with 15+ properties, `AgentStatuses` constants, `BlazorDataModelLoader` extension. **Done.**
- `FreeServicesHub.App.DataAccess.Agents.cs` — Full `GetAgents`/`SaveAgents`/`DeleteAgents` with TenantId scoping, admin/non-admin filtering, soft-delete, SignalR broadcast. **Done.**
- `FreeServicesHub.App.DataAccess.Heartbeats.cs` — `SaveHeartbeat` with CPU/memory threshold detection, `GetHeartbeats` with time-window cutoff, `PruneHeartbeats` with configurable retention. **Done.**
- `FreeServicesHub.App.DataAccess.ApiKeys.cs` — SHA-256 hashed registration keys, one-time-use validation, API client token generation/revocation. **Done.**

**What's missing:** There is no `HubJob` anywhere. No EF entity, no DTO, no DataAccess methods, no controller endpoints. The Agent has no way to ask "do I have any work to do?" and the Hub has no way to say "run this script on that server."

The pattern we follow is identical to the Agent pipeline — same naming conventions, same CRUD signatures, same SignalR broadcast structure. We're not inventing anything new. We're repeating a proven pattern for a new entity.

### P1-1B: DTO Class (HubJob)

**Why:** The EF entity lives in `EFModels` and maps directly to the database schema. But the rest of the application (DataAccess, Client, Agent) should never touch EF entities directly — they work with Data Transfer Objects in the `DataObjects` project. This separation is a core FreeCRM pattern: the DTO extends `ActionResponseObject` (giving it `ActionResponse` with `Result` + `Messages` for error handling), while the EF entity is a plain POCO. The existing `DataObjects.Agent` class follows this exact pattern — it has `ActionResponseObject` inheritance plus denormalized display fields like `AddedBy` (which shows a display name, not a raw user ID).

**What's lacking today:** The Agent DTO exists (`DataObjects.Agent`), the heartbeat DTO exists (`DataObjects.AgentHeartbeat`), the API key DTOs exist (`DataObjects.RegistrationKey`, `DataObjects.ApiClientToken`). There is no job DTO. Without it, neither the Client Blazor pages nor the Agent Worker Service can serialize/deserialize job data.

### P1-3: DataAccess CRUD

**Why:** The DataAccess layer is where business logic lives in FreeCRM. It sits between the EF database and the API controllers. Every entity follows the Three-Endpoint CRUD pattern: `GetMany(List<Guid>? Ids, Guid TenantId)`, `SaveMany(List<T> Items)`, `DeleteMany(List<Guid>? Ids)`. This is a non-negotiable project convention documented in [007_patterns.crud_api.md](007_patterns.crud_api.md) — we don't use standard REST verbs.

The existing `FreeServicesHub.App.DataAccess.Agents.cs` is 150+ lines with tenant-scoped filtering, admin/non-admin visibility, soft-delete handling, field mapping, string length enforcement, and SignalR broadcast on save. The job DataAccess mirrors this line-for-line, but adds one new method: `GetJobsForAgent(Guid AgentId, Guid TenantId)` — this is the Agent-facing query that returns only jobs in `Queued` or `Assigned` status for a specific agent, ordered by priority then creation date.

**What's lacking today:** The `IDataAccess` partial interface has methods for `Agents`, `Heartbeats`, `RegistrationKeys`, and `ApiClientTokens`. There is no way to query, create, or delete jobs. The Agent worker service has no API to call.

**File:** `FreeServicesHub/FreeServicesHub.DataAccess/FreeServicesHub.App.DataAccess.Jobs.cs` — **NEW**

```csharp
namespace FreeServicesHub;

public partial interface IDataAccess
{
    Task<List<DataObjects.HubJob>> GetJobs(List<Guid>? Ids, Guid TenantId, DataObjects.User? CurrentUser = null);
    Task<List<DataObjects.HubJob>> SaveJobs(List<DataObjects.HubJob> Items, DataObjects.User? CurrentUser = null);
    Task<DataObjects.BooleanResponse> DeleteJobs(List<Guid>? Ids, DataObjects.User? CurrentUser = null);
    Task<List<DataObjects.HubJob>> GetJobsForAgent(Guid AgentId, Guid TenantId);
}

public partial class DataAccess
{
    public async Task<List<DataObjects.HubJob>> GetJobs(
        List<Guid>? Ids, Guid TenantId, DataObjects.User? CurrentUser = null)
    {
        List<DataObjects.HubJob> output = new();

        IQueryable<EFModels.EFModels.HubJob> query = data.HubJobs
            .Where(x => x.TenantId == TenantId);

        if (Ids != null && Ids.Any()) {
            query = query.Where(x => Ids.Contains(x.HubJobId));
        }

        List<EFModels.EFModels.HubJob> recs = await query
            .OrderByDescending(x => x.Created)
            .ToListAsync();

        List<Guid> agentIds = recs
            .Where(x => x.AssignedAgentId.HasValue)
            .Select(x => x.AssignedAgentId!.Value)
            .Distinct().ToList();

        List<EFModels.EFModels.Agent> agents = agentIds.Any()
            ? await data.Agents.Where(a => agentIds.Contains(a.AgentId)).ToListAsync()
            : new();

        foreach (EFModels.EFModels.HubJob rec in recs) {
            output.Add(MapJobToDto(rec, agents));
        }

        return output;
    }

    public async Task<List<DataObjects.HubJob>> GetJobsForAgent(Guid AgentId, Guid TenantId)
    {
        List<EFModels.EFModels.HubJob> recs = await data.HubJobs
            .Where(x => x.TenantId == TenantId
                && x.AssignedAgentId == AgentId
                && (x.Status == DataObjects.HubJobStatuses.Queued
                    || x.Status == DataObjects.HubJobStatuses.Assigned))
            .OrderBy(x => x.Priority)
            .ThenBy(x => x.Created)
            .ToListAsync();

        return recs.Select(r => MapJobToDto(r, null)).ToList();
    }

    public async Task<List<DataObjects.HubJob>> SaveJobs(
        List<DataObjects.HubJob> Items, DataObjects.User? CurrentUser = null)
    {
        List<DataObjects.HubJob> output = new();
        foreach (DataObjects.HubJob item in Items) {
            output.Add(await SaveJob(item, CurrentUser));
        }
        return output;
    }

    private async Task<DataObjects.HubJob> SaveJob(
        DataObjects.HubJob job, DataObjects.User? CurrentUser = null)
    {
        DataObjects.HubJob output = job;
        output.ActionResponse = GetNewActionResponse();

        bool newRecord = false;
        DateTime now = DateTime.UtcNow;

        EFModels.EFModels.HubJob? rec = await data.HubJobs
            .FirstOrDefaultAsync(x => x.HubJobId == output.HubJobId);

        if (rec == null) {
            if (output.HubJobId == Guid.Empty) {
                newRecord = true;
                output.HubJobId = Guid.NewGuid();
                rec = new EFModels.EFModels.HubJob {
                    HubJobId = output.HubJobId,
                    TenantId = output.TenantId,
                    Created = now,
                    CreatedBy = CurrentUserIdString(CurrentUser),
                };
            } else {
                output.ActionResponse.Messages.Add(
                    "Job '" + output.HubJobId.ToString() + "' Not Found");
                return output;
            }
        }

        rec.AssignedAgentId = output.AssignedAgentId;
        rec.JobType = MaxStringLength(output.JobType, 100);
        rec.JobPayloadJson = output.JobPayloadJson;
        rec.Status = output.Status;
        rec.Priority = output.Priority;
        rec.ScheduledAt = output.ScheduledAt;
        rec.StartedAt = output.StartedAt;
        rec.CompletedAt = output.CompletedAt;
        rec.ResultJson = output.ResultJson;
        rec.ErrorMessage = MaxStringLength(output.ErrorMessage, 2000);
        rec.RetryCount = output.RetryCount;
        rec.MaxRetries = output.MaxRetries;
        rec.LastModified = now;
        rec.LastModifiedBy = CurrentUserIdString(CurrentUser);

        try {
            if (newRecord) {
                await data.HubJobs.AddAsync(rec);
            }
            await data.SaveChangesAsync();
            output.ActionResponse.Result = true;

            string updateType = output.Status == DataObjects.HubJobStatuses.Completed
                ? DataObjects.SignalRUpdateType.JobCompleted
                : DataObjects.SignalRUpdateType.JobUpdated;

            await SignalRUpdate(new DataObjects.SignalRUpdate {
                TenantId = output.TenantId,
                ItemId = output.HubJobId,
                UpdateType = updateType,
                Message = newRecord ? "Created" : "Updated",
                UserId = CurrentUserId(CurrentUser),
                Object = output,
            });
        } catch (Exception ex) {
            output.ActionResponse.Messages.Add(
                "Error Saving Job " + output.HubJobId.ToString());
            output.ActionResponse.Messages.AddRange(RecurseException(ex));
        }

        return output;
    }

    public async Task<DataObjects.BooleanResponse> DeleteJobs(
        List<Guid>? Ids, DataObjects.User? CurrentUser = null)
    {
        DataObjects.BooleanResponse output = new();
        if (Ids == null || !Ids.Any()) {
            output.Messages.Add("No Job Ids provided.");
            return output;
        }

        try {
            List<EFModels.EFModels.HubJob> recs = await data.HubJobs
                .Where(x => Ids.Contains(x.HubJobId)).ToListAsync();

            if (recs.Any()) {
                Guid tenantId = recs.First().TenantId;
                data.HubJobs.RemoveRange(recs);
                await data.SaveChangesAsync();
                output.Result = true;

                await SignalRUpdate(new DataObjects.SignalRUpdate {
                    TenantId = tenantId,
                    UpdateType = DataObjects.SignalRUpdateType.JobUpdated,
                    Message = "Deleted " + recs.Count + " job(s)",
                    UserId = CurrentUserId(CurrentUser),
                });
            }
        } catch (Exception ex) {
            output.Messages.Add("Error Deleting Jobs");
            output.Messages.AddRange(RecurseException(ex));
        }

        return output;
    }

    private static DataObjects.HubJob MapJobToDto(
        EFModels.EFModels.HubJob rec, List<EFModels.EFModels.Agent>? agents)
    {
        string agentName = string.Empty;
        if (rec.AssignedAgentId.HasValue && agents != null) {
            agentName = agents
                .FirstOrDefault(a => a.AgentId == rec.AssignedAgentId.Value)?.Name
                ?? string.Empty;
        }

        return new DataObjects.HubJob {
            ActionResponse = new DataObjects.ActionResponse { Result = true },
            HubJobId = rec.HubJobId,
            TenantId = rec.TenantId,
            AssignedAgentId = rec.AssignedAgentId,
            JobType = rec.JobType ?? string.Empty,
            JobPayloadJson = rec.JobPayloadJson ?? string.Empty,
            Status = rec.Status ?? string.Empty,
            Priority = rec.Priority,
            ScheduledAt = rec.ScheduledAt,
            StartedAt = rec.StartedAt,
            CompletedAt = rec.CompletedAt,
            ResultJson = rec.ResultJson ?? string.Empty,
            ErrorMessage = rec.ErrorMessage ?? string.Empty,
            RetryCount = rec.RetryCount,
            MaxRetries = rec.MaxRetries,
            Created = rec.Created,
            CreatedBy = rec.CreatedBy ?? string.Empty,
            LastModified = rec.LastModified,
            LastModifiedBy = rec.LastModifiedBy ?? string.Empty,
            AssignedAgentName = agentName,
        };
    }
}
```

### P1-5b: Agent API Endpoints

**Why:** The DataController endpoints are for **authenticated users** (cookie/session auth from the Blazor UI). But the Agent is a headless Windows Service that authenticates with a Bearer token, not a browser cookie. The existing `FreeServicesHub.App.API.cs` already has agent-facing endpoints (heartbeat, registration, settings) that extract `AgentId` and `AgentTenantId` from `HttpContext.Items` — set by the `ApiKeyMiddleware`. We need two new endpoints on this same controller: one for the Agent to fetch its assigned jobs, and one for the Agent to report job completion.

## Phase 2: Agent Hardening & Security

### Why Phase 2 Exists

Phase 1 gives us the data plumbing — a `HubJob` table, DTOs, CRUD methods, and API endpoints. But the data just sits there. Nobody fetches it. The Agent worker service currently has one loop: collect system snapshots and POST heartbeats. It has no awareness that jobs exist.

Additionally, the Opus code exploration uncovered one real security vulnerability: `AgentMonitorService.PollAndBroadcast()` broadcasts agent status changes to a **global** SignalR group called `"AgentMonitor"`. In a multi-tenant deployment, Tenant B's dashboard users would see Tenant A's agent data. This was missed in earlier reviews because single-tenant testing masks the bug completely.

Phase 2 has two goals:
1. **Wire the Agent to the job queue** — add a parallel polling loop that fetches assigned jobs, executes them locally, and reports results.
2. **Fix the tenant leak** — partition the `AgentMonitor` group by TenantId.

```ascii
  BEFORE (Phase 2):                         AFTER (Phase 2):
  ═════════════════                         ════════════════
  Agent:                                    Agent:
    [heartbeat loop] ──► Hub                  [heartbeat loop] ──► Hub
    (that's it)                               [job poll loop] ◄──► Hub
                                                ├── FetchMyJobs()
  AgentMonitorService:                          ├── ProcessJob()
    broadcasts to "AgentMonitor"                └── ReportJobStatus()
    (global group, all tenants)
                                            AgentMonitorService:
                                              broadcasts to "AgentMonitor_{TenantId}"
                                              (tenant-scoped, no leakage)
```

### P2-1: AgentMonitorService Tenant Scoping

**Why:** This is the **one real security bug** found during the code exploration. The `AgentMonitorService` is a background service that polls all agents every 5 seconds, detects stale/offline agents, and broadcasts status changes via SignalR. The problem: it loads agents with `GetAgents(null, Guid.Empty, null)` — passing `Guid.Empty` as the TenantId — and then broadcasts to `Clients.Group("AgentMonitor")`, a single global group. In a multi-tenant deployment, all dashboard users join the same group regardless of their tenant. Result: Tenant A sees Tenant B's servers, hostnames, CPU usage, and online/offline status.

**What's lacking today:** The `MonitorGroup` constant is hardcoded to `"AgentMonitor"`. There's no per-tenant partitioning. The fix: query distinct tenant IDs from the Agents table, loop per-tenant, and broadcast to `"AgentMonitor_{tenantId}"`. The client-side also needs to join the correct tenant-scoped group.

**File:** `FreeServicesHub/FreeServicesHub/FreeServicesHub.App.AgentMonitorService.cs` — **MODIFY**

Replace the `PollAndBroadcast` method. Key change: iterate per-tenant instead of broadcasting globally.

```csharp
    private async Task PollAndBroadcast(CancellationToken StoppingToken)
    {
        using IServiceScope scope = _serviceProvider.CreateScope();
        IDataAccess da = scope.ServiceProvider.GetRequiredService<IDataAccess>();
        IConfigurationHelper config = scope.ServiceProvider.GetRequiredService<IConfigurationHelper>();
        var dbContext = scope.ServiceProvider.GetRequiredService<EFModels.EFModels.EFDataModel>();

        int staleThreshold = config.AgentStaleThresholdSeconds;
        DateTime staleEdge = DateTime.UtcNow.AddSeconds(-staleThreshold);

        // Get distinct tenant IDs that have agents
        List<Guid> tenantIds = await dbContext.Agents
            .Where(a => !a.Deleted)
            .Select(a => a.TenantId)
            .Distinct()
            .ToListAsync();

        foreach (Guid tenantId in tenantIds) {
            List<DataObjects.Agent> agents = await da.GetAgents(null, tenantId, null);
            List<DataObjects.Agent> changedAgents = new();

            foreach (DataObjects.Agent agent in agents) {
                string previousStatus = agent.Status ?? DataObjects.AgentStatuses.Offline;
                string newStatus = previousStatus;

                if (agent.LastHeartbeat.HasValue && agent.LastHeartbeat.Value < staleEdge) {
                    newStatus = previousStatus == DataObjects.AgentStatuses.Online
                        ? DataObjects.AgentStatuses.Stale
                        : DataObjects.AgentStatuses.Offline;
                }

                bool changed = false;
                if (_cachedStatuses.TryGetValue(agent.AgentId, out string? cached)) {
                    changed = cached != newStatus;
                } else {
                    changed = true;
                }

                _cachedStatuses[agent.AgentId] = newStatus;

                if (changed) {
                    agent.Status = newStatus;
                    changedAgents.Add(agent);
                }
            }

            // Broadcast to tenant-scoped group
            string tenantGroup = $"AgentMonitor_{tenantId}";

            if (changedAgents.Count > 0) {
                DataObjects.SignalRUpdate changeUpdate = new() {
                    TenantId = tenantId,
                    UpdateType = DataObjects.SignalRUpdateType.AgentStatusChanged,
                    Message = $"{changedAgents.Count} agent(s) status changed",
                    Object = changedAgents,
                };
                await _hubContext.Clients.Group(tenantGroup).SignalRUpdate(changeUpdate);
            }

            DataObjects.SignalRUpdate heartbeatUpdate = new() {
                TenantId = tenantId,
                UpdateType = DataObjects.SignalRUpdateType.AgentHeartbeat,
                Message = changedAgents.Count > 0
                    ? $"{changedAgents.Count} change(s) across {agents.Count} agents"
                    : $"Checked {agents.Count} agents — no changes",
                Object = agents,
            };
            await _hubContext.Clients.Group(tenantGroup).SignalRUpdate(heartbeatUpdate);
        }
    }
```

### P2-2: Agent Job Polling Loop

**Why:** This is the core feature of the entire action plan — the thing that turns FreeServicesHub from a monitoring dashboard into a command-and-control platform. Today the Agent's `ExecuteAsync` method runs a single `while` loop: sleep → collect snapshot → POST heartbeat → repeat. We need a second parallel loop: sleep → GET `/api/agent/jobs` → for each job: mark Running, execute locally, mark Completed/Failed, POST result → repeat.

The two loops must be independent. If job execution takes 5 minutes (e.g., collecting a large log file), the heartbeat loop must continue sending snapshots every 30 seconds. If the Hub is temporarily unreachable, both loops use exponential backoff independently.

**What's lacking today:** The Agent is a one-trick pony — it reports metrics but can't be told to do anything. The `AgentWorkerService.cs` has no reference to `HubJob`, no HTTP GET for jobs, and no local execution dispatch.

```csharp
    private async Task JobPollLoop(CancellationToken stoppingToken)
    {
        int pollIntervalSeconds = 10;
        int consecutiveErrors = 0;

        // Wait for initial registration to complete
        while (!_registered && !stoppingToken.IsCancellationRequested) {
            await Task.Delay(TimeSpan.FromSeconds(5), stoppingToken);
        }

        _logger.LogInformation("Job poll loop started");

        while (!stoppingToken.IsCancellationRequested) {
            try {
                List<DataObjects.HubJob> jobs = await FetchMyJobs();

                if (jobs.Count > 0) {
                    _logger.LogInformation("Received {Count} job(s) to process", jobs.Count);
                }

                foreach (DataObjects.HubJob job in jobs) {
                    if (stoppingToken.IsCancellationRequested) break;
                    await ProcessJob(job);
                }

                consecutiveErrors = 0;
            } catch (HttpRequestException ex) {
                consecutiveErrors++;
                _logger.LogWarning(ex, "Job poll failed (attempt {Count})", consecutiveErrors);
            } catch (Exception ex) {
                consecutiveErrors++;
                _logger.LogError(ex, "Unexpected job poll error (attempt {Count})", consecutiveErrors);
            }

            int backoff = Math.Min(consecutiveErrors + 1, 6);
            await Task.Delay(
                TimeSpan.FromSeconds(pollIntervalSeconds * backoff),
                stoppingToken);
        }
    }

    private async Task<List<DataObjects.HubJob>> FetchMyJobs()
    {
        using HttpClient client = new();
        client.DefaultRequestHeaders.Authorization =
            new System.Net.Http.Headers.AuthenticationHeaderValue("Bearer", _options.ApiClientToken);

        HttpResponseMessage response = await client.GetAsync(
            $"{_options.HubUrl}/api/agent/jobs");
        response.EnsureSuccessStatusCode();

        return await response.Content.ReadFromJsonAsync<List<DataObjects.HubJob>>()
            ?? new();
    }

    private async Task ProcessJob(DataObjects.HubJob job)
    {
        _logger.LogInformation("Processing job {JobId} type={Type}",
            job.HubJobId, job.JobType);

        // Mark as Running
        job.Status = "Running";
        job.StartedAt = DateTime.UtcNow;
        await ReportJobStatus(job);

        try {
            string result = await ExecuteJobLocally(job);
            job.Status = "Completed";
            job.ResultJson = result;
            job.CompletedAt = DateTime.UtcNow;
            _logger.LogInformation("Job {JobId} completed successfully", job.HubJobId);
        } catch (Exception ex) {
            job.RetryCount++;
            job.ErrorMessage = ex.Message;

            if (job.RetryCount >= job.MaxRetries) {
                job.Status = "Failed";
                _logger.LogError(ex, "Job {JobId} failed permanently after {Retries} retries",
                    job.HubJobId, job.RetryCount);
            } else {
                job.Status = "Queued";
                _logger.LogWarning(ex, "Job {JobId} failed, re-queuing (attempt {Count}/{Max})",
                    job.HubJobId, job.RetryCount, job.MaxRetries);
            }
        }

        await ReportJobStatus(job);
    }

    private async Task<string> ExecuteJobLocally(DataObjects.HubJob job)
    {
        return job.JobType switch {
            "CollectLogs" => await CollectLogs(job.JobPayloadJson),
            "RestartService" => await RestartWindowsService(job.JobPayloadJson),
            "RunScript" => await RunScript(job.JobPayloadJson),
            _ => throw new NotSupportedException($"Unknown job type: {job.JobType}")
        };
    }

    private async Task<string> CollectLogs(string payloadJson)
    {
        // Parse payload for log path, line count, etc.
        await Task.Delay(100); // Placeholder for actual file read
        return "{\"status\":\"collected\",\"lines\":0}";
    }

    private async Task<string> RestartWindowsService(string payloadJson)
    {
        // Parse payload for service name
        await Task.Delay(100); // Placeholder for sc.exe restart
        return "{\"status\":\"restarted\"}";
    }

    private async Task<string> RunScript(string payloadJson)
    {
        // Parse payload for script content/path
        await Task.Delay(100); // Placeholder for process execution
        return "{\"status\":\"executed\",\"exitCode\":0}";
    }

    private async Task ReportJobStatus(DataObjects.HubJob job)
    {
        try {
            using HttpClient client = new();
            client.DefaultRequestHeaders.Authorization =
                new System.Net.Http.Headers.AuthenticationHeaderValue("Bearer", _options.ApiClientToken);

            await client.PostAsJsonAsync(
                $"{_options.HubUrl}/api/agent/jobs/complete", job);
        } catch (Exception ex) {
            _logger.LogWarning(ex, "Failed to report job {JobId} status", job.HubJobId);
        }
    }
```

```csharp
    private bool _wizardVerifying = false;
    private bool _wizardVerified = false;
    private string _selectedKeyPrefix = string.Empty;
    private DateTime _keyGeneratedAt;

    private async Task VerifyAgentConnection()
    {
        _wizardVerifying = true;
        _wizardVerified = false;
        StateHasChanged();

        int maxAttempts = 12; // 12 * 5s = 60 seconds
        for (int i = 0; i < maxAttempts; i++) {
            var agents = await Http.GetFromJsonAsync<List<DataObjects.Agent>>(
                $"api/data/agents?tenantId={Model.TenantId}") ?? new();

            var match = agents.FirstOrDefault(a =>
                a.RegisteredBy != null
                && a.RegisteredBy.Contains(_selectedKeyPrefix)
                && a.RegisteredAt > _keyGeneratedAt);

            if (match != null) {
                _wizardVerified = true;
                _wizardVerifying = false;
                _statusMessage = $"Agent '{match.Name}' connected successfully!";
                _statusAlertClass = "alert-success";
                StateHasChanged();
                return;
            }

            await Task.Delay(5000);
            StateHasChanged(); // Update "checking..." indicator
        }

        _wizardVerifying = false;
        _statusMessage = "Agent has not connected yet. It may still be starting up — check the Agent Settings page.";
        _statusAlertClass = "alert-warning";
        StateHasChanged();
    }
```
